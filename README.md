# Programming Assignment 3 - Highest Value Longest Common Subsequence

## Overview
This project computes a highest-value common subsequence between two strings `A` and `B`, where each alphabet character has a nonnegative integer value. The solver outputs:
1. the maximum total value of a common subsequence, and
2. one optimal subsequence achieving that value.

## Repository Structure
- `src/main.py` - Main entry point. Supports direct CLI mode and interactive case-runner menu mode.
- `src/parser.py` - Input parsing and validation.
- `src/hvlcs.py` - Dynamic programming HVLCS solver and subsequence reconstruction.
- `src/hvlcs_empirical_comparison.py` - Generates random `hvlcs1..10` inputs, times runtime, writes outputs, and saves runtime artifacts.
- `data/inputs/` - Includes `example.in`, `single_match.in`, `no_match.in`, `repeated_chars.in`, and generated `hvlcs1.in` to `hvlcs10.in`.
- `data/outputs/` - Includes expected outputs for the above cases: `example.out`, `single_match.out`, `no_match.out`, `repeated_chars.out`, and `hvlcs1.out` to `hvlcs10.out`.
- `results/runtime_data.csv` - Runtime measurements generated by the empirical comparison script.
- `results/hvlcs_runtime_vs_input.png` - Runtime plot generated by the empirical comparison script.

## Input / Output Format
### Input
Input files use:

```text
K
x1 v1
x2 v2
...
xK vK
A
B
```

- `K` = number of characters in the alphabet
- each of the next `K` lines contains one character and its assigned value
- `A` = first string
- `B` = second string

### Output
The program prints:

```text
<maximum_value>
<one_optimal_subsequence>
```

The first line is the maximum value of a common subsequence of `A` and `B`.  
The second line is one optimal subsequence that achieves that value.

## How to Run
### Requirements
- Python 3.x
- No external packages required for `src/main.py`

### Direct Mode (single input file)
Run from the repository root:

```text
python src/main.py data/inputs/example.in
```

### Menu Mode (interactive case runner)
Run from the repository root:

```text
python src/main.py
```

This opens a menu with:
1. Run single case
2. Run all cases
3. Exit

## Case Runner Menu
The menu auto-discovers and pairs cases by matching basename:
- input: `data/inputs/<name>.in`
- expected output: `data/outputs/<name>.out`

Only matched pairs are listed/runnable.

### Validation rule
- `value match` is the required pass/fail criterion.
- `subsequence exact match` is informational only.

This is useful because multiple distinct subsequences can have the same optimal value.

## Assumptions
- Input files follow the required format.
- Character values are nonnegative integers.
- Strings use only symbols listed in the alphabet section.
- If multiple optimal subsequences exist, the solver may return any one of them.
